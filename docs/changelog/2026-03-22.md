## Day Zero: The Six Judges

Today we ran Coinnect through a gauntlet. Six expert perspectives — architecture, economics, law, UX, security, and launch readiness — each examined the project independently and returned honest scores.

**The verdict: 5.7/10 average.** Not bad for a solo-built MVP. Not ready for launch either.

### What we found

The product is surprisingly mature. 21 providers, 15 languages, MCP server, open data API, rate history with sparklines, a whitepaper that reads like it was written by someone who actually cares about the problem. The routing engine — dual Dijkstra with deduplication — is elegant and correct.

But underneath, there were bombs waiting to go off.

### What we fixed today

**Security (the scary stuff):**
- The admin key was hardcoded in the source code: `"coinnect-admin-2026"`. Anyone who read the repo could access `/admin/stats` and see every search ever made. Fixed: now requires `COINNECT_ADMIN_KEY` environment variable, no fallback.
- The `/rates/{id}` snapshot page was vulnerable to XSS — provider names from the database were inserted directly into HTML without escaping. Fixed: all dynamic values now pass through `html.escape()`.
- SQLite was running without WAL (Write-Ahead Logging) mode across 4 uvicorn workers. Under load, this means `database is locked` errors and potential corruption. Fixed: WAL mode + 5-second busy timeout on all three database modules.

**Data freshness:**
- Exchange rate caches in Wise, Yellow Card, and remittance adapters had no TTL. Once fetched, a rate lived forever in memory until restart. If an API returned a bad rate at 3am, users saw that bad rate all day. Fixed: 5-minute TTL on all rate caches, with graceful fallback to stale data if the API is temporarily down.

**Missing corridors:**
- Western Union and MoneyGram only had outbound corridors (USD/EUR → MXN, NGN, etc.) but no reverse. A user searching MXN → USD wouldn't see WU or MG at all. Fixed: added 10 reverse corridors for WU and 8 for MG, covering the most important remittance-receiving countries sending back.

**UX:**
- "No Tracking" button in the footer now says "Disable Tracking" across all 15 languages. The old text was descriptive; the new text is actionable.
- Health endpoint no longer exposes the server's filesystem path (`history_db` field removed).
- Suggestion and vote endpoints now have IP-based rate limiting (5 suggestions/hr, 20 votes/hr per IP) to prevent bot spam.

**Documentation:**
- OpenAPI spec updated to include all 12 API endpoints (was missing 7 of them).

### What the judges said that hurt the most

The economic judge scored us 4.5/10. The core insight: *"The donation model doesn't work for invisible infrastructure. Wikipedia can do it because 1.5 billion people visit every month. A remittance comparator won't have that scale. Plan for zero donation income."*

The recommendation that stuck: **"5 Pro API clients at $100/month changes everything."** That's $6,000/year — covers the founder's salary with the current burn rate. No ads. No affiliates. No compromise.

### What's next

Tomorrow night: launch. The service is live, the fixes are deployed, the GitHub repo is public. The question isn't whether the product works — it does. The question is whether anyone finds it.

The six judges agreed on one thing: **the product is ready. The infrastructure needs one more day.** Monitoring, backups, and a clean deploy are all that remain.
