# Coinnect — Governance & Participation

> Coinnect is the reference implementation of the **Money Routing Protocol (MRP)**.
> It is a non-profit, open-source project. No affiliate fees. No investors. No custody.

---

## Who maintains this

**Miguel Valencia** created Coinnect during an unemployment period in early 2026 as a response to a simple observation: there is no neutral, open tool that shows you the cheapest way to send money globally. He continues to maintain it.

This is not a startup. There are no VCs, no exit strategy, no affiliate kickbacks. Every route shown is ranked by cost to the sender — not by what earns Coinnect money.

If this project has saved you money, or if you work at an institution that cares about financial inclusion, open standards, or equitable access to global financial infrastructure — read on.

---

## How to participate

### As a developer

- Fork the repository, submit PRs
- Open issues for bugs, missing corridors, or API improvements
- Run your own MRP-compatible router (MIT license — do whatever you want)

### As an exchange or remittance provider

You can request to be listed. Requirements:
1. Public pricing documentation (we do not accept affiliate arrangements)
2. Corridor list with min/max send amounts and payment methods
3. A contact for rate update notifications

Email: **hello@coinnect.bot** · Subject: "Exchange listing request"

No fee. No exclusivity. Being listed does not guarantee favorable ranking — routes are always ranked by cost to the sender.

### As a researcher or institution

We welcome collaboration from:
- Financial inclusion researchers (World Bank, CGAP, BIS Innovation Hub)
- Open standards bodies (IETF, W3C, ISO TC68)
- Development finance institutions
- Foundations focused on financial access (Gates, Ford, Omidyar, Stellar)
- Academic groups studying remittance markets, payment systems, or open infrastructure

If you're working on related problems — cross-border payment standardization, remittance cost reduction, financial inclusion measurement — we want to talk.

**Contact:** hello@coinnect.bot

### As a funder

Coinnect operates at near-zero cost today (one VPS, open APIs). As it grows, it will need:
- Maintained infrastructure
- Legal entity / fiscal sponsor
- A sustainable salary for the founder/maintainer
- Potentially: a small working group stipend

We are actively seeking:
- **Foundation grants** (EU NGI, NLnet, Gates Foundation, Ford Foundation)
- **Institutional partnerships** (development banks, UN agencies)
- **Academic collaboration** (joint grants, research partnerships)

We are NOT seeking:
- VC investment
- Commercial partnerships that create conflicts of interest
- Affiliate arrangements with listed providers

## Decision-making

For now, decisions are made by the maintainer (Miguel) with public input via GitHub issues and discussions.

As the project grows, governance will evolve toward a Working Group model (IETF-style rough consensus). The goal is to make MRP a community standard that outlives any single implementation.

Protocol changes that affect the API contract or Edge schema will be:
1. Proposed as a GitHub issue with rationale
2. Open for comment for at least 30 days
3. Adopted by rough consensus of active contributors
